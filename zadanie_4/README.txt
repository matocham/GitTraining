Praca z gałęziami

1. Utwórz nowe repozytorium
2. Stwórz kilka rewizji w gałęzi master
3. Stwórz gałąź test_1 oraz dodaj nowe pliki, stwórz co najmniej 2 nowe rewizje
4. Wróć do gałęzi master
5. Zmień nazwę gałęzi test_1 na feature_1
6. Scal gałąź feature_1 z gałęzią master korzystając z polecenia merge
7. Sprawdź historię projektu za pomocą polecenia git log z opcją --graph
8. Wycofaj operację merge wykorzystując polecenie git reset
9. Scal gałąź feature_1  z gałęzią master korzystając z polecenia rebase.
10. Sprawdź historię projektu za pomocą polecenia git log z opcją graph. Porównaj historię rewizji
11. Usuń gałąź feature_1 za pomocą polecenia git branch –d
