Zmiana historii repozytorium

1. W repozytorium z Zadania 1 zmodyfikuj dowolny plik oraz stwórz nową rewizję
2. Wycofaj wprowadzone zmiany za pomocą polecenia git revert. Wykorzystaj identyfikator HEAD
3. Wyświetl historię repozytorium
4. Za pomocą polecenia git log wyświetl wszystkie rewizje, w których został zmodyfikowany wybrany plik
5. Wykorzystując polecenie git checkout przywróć stan pliku A/file1.txt sprzed ostatniej modyfikacji. Wykorzystaj identyfikator HEAD
6. Zatwierdź wprowadzone zmiany
7. Zmodyfikuj dwa dowolne pliki oraz dodaj jeden nowy. Dodaj nowy oraz jeden ze zmodyfikowanych plików do poczekalni. Sprawdź stan repozytorium
8. Przetestuj różne wersje polecenia reset (soft/mixed/hard) oraz sprawdź zmiany spowodowane w repozytorium przez każde z nich za pomocą poleceń git status oraz git log.
