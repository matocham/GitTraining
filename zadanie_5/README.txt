Rozwiązywanie konfliktów

1. W repozytorium z Zadania 1 utwórz nową gałąź o nazwie topic_A korzystając z polecenia git checkout -b
2. Zmodyfikuj pliki A/file1.txt oraz B/file2.txt
3. Zatwierdź wprowadzone zmiany oraz przejdź do gałęzi master.
4. Zmodyfikuj te same pliki wprowadzając inne zmiany w tych samych liniach
5. Scal gałęzie topic_A oraz master jedną z wybranych metod merge lub rebase
6. Rozwiąż konflikty powstałe podczas scalania
7. Zakończ operację scalania w sposób odpowiedni do wybranej metody (git commit lub git rebase --continue)
